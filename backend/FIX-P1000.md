# Fix P1000 Authentication Failed

Error means PostgreSQL rejected the username/password in `backend\.env`.

## Fix A — Reset `learniq` user in pgAdmin (keep current .env)

1. Open **pgAdmin 4**
2. Connect to **PostgreSQL 18** (use your **postgres** password)
3. Click database **`postgres`** (the default one, not learniq)
4. **Tools** → **Query Tool**
5. Paste and run:

```sql
ALTER ROLE learniq WITH LOGIN PASSWORD 'learniq_secret';
```

If you get **role "learniq" does not exist**, run instead:

```sql
CREATE ROLE learniq WITH LOGIN PASSWORD 'learniq_secret';
CREATE DATABASE learniq OWNER learniq;
```

6. Confirm `backend\.env` has exactly:

```
DATABASE_URL="postgresql://learniq:learniq_secret@localhost:5432/learniq"
```

7. Test in PowerShell:

```powershell
cd "c:\New folder (3)\backend"
npx prisma db push
```

---

## Fix B — Use `postgres` user (easiest if pgAdmin works)

1. In pgAdmin **Query Tool** on database `postgres`, run:

```sql
CREATE DATABASE learniq;
```

(Ignore error if database already exists.)

2. Edit `backend\.env` — replace YOUR_PASSWORD with the password you use to open pgAdmin:

```env
DATABASE_URL="postgresql://postgres:YOUR_PASSWORD@localhost:5432/learniq"
```

**Important:** If password has special characters (`@`, `#`, `%`), URL-encode them or wrap carefully.

Examples:
- Password `admin123` → `postgresql://postgres:admin123@localhost:5432/learniq`
- Password `pass@word` → `postgresql://postgres:pass%40word@localhost:5432/learniq`

3. Run:

```powershell
npx prisma db push
npm run db:seed
```

---

## Fix C — Password has special characters

Use Prisma format with encoded password, or set in pgAdmin a simple password for testing:

```sql
ALTER ROLE learniq WITH PASSWORD 'learniq_secret';
```

Only letters and numbers in password avoids encoding issues.

## Verify before prisma

Install pg client once:

```powershell
cd "c:\New folder (3)\backend"
npm install pg
node scripts/test-db-connection.js
```

If that prints **Connected successfully**, then `npx prisma db push` will work.
